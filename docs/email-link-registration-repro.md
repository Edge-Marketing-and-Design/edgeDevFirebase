# Email-Link Registration Repro Steps

These steps verify `registerUser(..., "emailLink")` for:
1. happy path
2. failure path (missing stored email context)

## Prereqs
- Firebase emulators running for auth/firestore/functions.
- A `staged-users/{registrationCode}` doc exists with:
  - `userId: ""`
  - `isTemplate: false` (or valid template fields if testing template mode)

## Happy path
1. Create an `EdgeFirebase` instance connected to emulator config.
2. Call:
   ```ts
   await edgeFirebase.sendEmailLink("person@example.com", "http://localhost:3000/register?mode=signIn")
   ```
3. Open email-link URL generated by emulator (Auth emulator UI / logs).
4. On callback page, call:
   ```ts
   const result = await edgeFirebase.registerUser(
     {
       registrationCode: "<REGISTRATION_CODE>",
       email: "person@example.com",
       meta: { firstName: "Person" }
     },
     "emailLink"
   )
   ```
5. Expect:
   - `result.success === true`
   - authenticated Firebase user exists (`edgeFirebase.user.uid` set after auth state update)
   - Firestore docs updated:
     - `users/{uid}` exists (created by backend sync)
     - `staged-users/{registrationCode}` has `userId == uid`, `uid == uid`
     - `rule-helpers/{uid}` exists with `edge-assignment-helper`

## Failure path: missing stored email context
1. Trigger callback without passing `email` and after clearing local storage key:
   ```ts
   localStorage.removeItem("edgeFirebase.emailLinkSignInEmail")
   ```
2. Call:
   ```ts
   const result = await edgeFirebase.signInWithEmailLink(undefined, window.location.href)
   ```
3. Expect:
   - `result.success === false`
   - `result.meta.code === "email-link/missing-email-context"`
   - `result.meta.firebaseCode === "auth/missing-email-context"`
   - user-usable message instructing to request a new link

## Optional failure path: invalid/expired link
1. Reuse an already-consumed link or tamper with `oobCode` in URL.
2. Call `edgeFirebase.signInWithEmailLink(email, tamperedLink)`.
3. Expect:
   - `success === false`
   - `meta.code === "email-link/invalid-or-used"` or `"email-link/expired"`
   - `meta.firebaseCode` contains the Firebase auth code.
